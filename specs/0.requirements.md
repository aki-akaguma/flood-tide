# Library Requirements

This document outlines the requirements for the `flood-tide` library.

## Core Functionality

The library provides utilities for parsing command-line arguments. It should be able to handle various common patterns for specifying options and arguments.

### 1. Option Parsing

The library must be able to parse the following types of options:

- **Short options:** A single hyphen followed by a single character (e.g., `-a`).
- **Combined short options:** A single hyphen followed by multiple characters, where each character is a separate option (e.g., `-abc` is equivalent to `-a -b -c`).
- **Long options:** Two hyphens followed by a descriptive name (e.g., `--verbose`).
- **Abbreviated long options:** A long option that can be specified by a prefix, as long as the prefix is unambiguous (e.g., `--verb` for `--verbose`).
- **Single-hyphen long options:** A single hyphen followed by a long option name (e.g., `-verbose`).

### 2. Option Arguments

The library must support options that take arguments. The following syntaxes should be supported:

- **Long option with space:** `--option value`
- **Long option with equals sign:** `--option=value`
- **Short option with space:** `-o value`
- **Short option with adjacent value:** `-ovalue`

The library should also support options that may have an optional argument.

### 3. Free Arguments

The library must be able to collect any arguments that are not options or option arguments. These are typically used for file paths or other positional arguments.

### 4. Subcommands

The library must support subcommands, allowing for different sets of options and arguments depending on the subcommand specified.

## Error Handling

The library should provide robust error handling. It should be able to detect and report the following types of errors:

- Invalid option
- Missing option
- Invalid option argument
- Unexpected option argument
- Missing option argument
- Unexpected argument
- Missing argument
- Invalid subcommand
- Missing subcommand
- Ambiguous option
- Ambiguous subcommand

The library should be configurable to either return the first error encountered or a collection of all errors.

## Platform Support

The library must be compatible with both `std` and `no_std` environments.

## Configuration

The library's features should be configurable at compile time to allow for smaller binary sizes when certain features are not needed.
