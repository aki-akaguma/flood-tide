# Design Document

This document outlines the design of the `flood-tide` library, a command-line argument parsing utility.

## 1. Architecture

The library follows a modular architecture, separating concerns into distinct components. The core components are:

- **`Lex`:** The lexical analyzer, responsible for tokenizing the input command-line arguments.
- **`Opt`:** Represents a single command-line option, defining its short and long names, and whether it accepts an argument.
- **`NameVal`:** A key-value pair representing a parsed option and its potential value.
- **`Tokens`:** A container for the results of the lexical analysis, including parsed options, free arguments, and any subcommands.
- **`OptParseError` and `OptParseErrors`:** Custom error types for handling parsing errors.

## 2. Data Structures

- **`Opt` struct:**

```rust
pub struct Opt<'a> {
    pub sho: u8,
    pub lon: &'a str,
    pub has: Arg,
    pub num: OptNum,
}
```

  This struct is the primary data structure for defining command-line options. It is designed to be created as a static array.

- **`sho_idx` array:**

  A sorted array of tuples `(u8, usize)` is used to quickly look up short options. The first element of the tuple is the short option character, and the second is the index into the `Opt` array.

- **`NameVal` struct:**

```rust
pub struct NameVal<'a> {
    pub opt: &'a Opt<'a>,
    pub val: Option<&'a str>,
    pub was_long: bool,
}
```

  This struct holds the result of parsing a single option. It contains a reference to the `Opt` definition, the option's value (if any), and a flag indicating whether the option was parsed as a long or short option.

- **`Tokens` struct:**

```rust
pub struct Tokens<'a> {
    pub namevals: Vec<NameVal<'a>>,
    pub double_m: bool,
    pub subcmd: Option<&'a str>,
    pub free: Vec<&'a str>,
}
```

  This struct aggregates the results of the entire parsing process.

## 3. Algorithms

### 3.1. Option Parsing

The `Lex::tokens_from` method is the main entry point for parsing. It iterates through the command-line arguments and dispatches to the appropriate parsing method based on the argument's format.

- **Long Options (`--`):** The `parse_long_name` method handles long options. It supports both `--option=value` and `--option value` syntaxes. If the `abbreviate` feature is enabled, it will attempt to find a unique long option that starts with the provided name.

- **Short Options (`-`):** The `parse_short_name` method handles short options. It supports combined short options (e.g., `-abc`) and options with adjacent values (e.g., `-ovalue`).

- **Single-Hyphen Long Options:** When the `long_only` feature is enabled, the `parse_long_only` method is used. It first attempts to parse the argument as a short option. If that fails, it tries to parse it as a long option.

### 3.2. Error Handling

The library can be configured to return either the first error encountered or a collection of all errors. This is controlled by the `single_error` feature flag.

The `OptParseError` enum defines the different types of parsing errors that can occur. The `Display` implementation for this enum provides user-friendly error messages.

### 3.3. Subcommand Parsing

If the `subcommand` feature is enabled, the `parse_subcmd` method is called after all options have been parsed. It checks if the first free argument is a valid subcommand. If the `abbreviate` feature is also enabled, it will attempt to find a unique subcommand that starts with the provided name.

## 4. Configuration

The library's functionality can be customized at compile time using feature flags. This allows users to reduce the binary size by only including the features they need. The available feature flags include:

- `no_std`: Enable `no_std` support.
- `single_error`: Return only the first parsing error.
- `option_argument`: Enable support for options with arguments.
- `argument`: Enable support for free arguments.
- `subcommand`: Enable support for subcommands.
- `abbreviate`: Enable support for abbreviated long options and subcommands.
- `long_only`: Treat single-hyphen arguments as long options.
- `stop_at_mm`: Stop parsing at `--`.
- `stop_at_free`: Stop parsing at the first free argument.
- `was_long`: Track whether an option was parsed as a long or short option.
- `optnum_u16`: Use `u16` for the option number type instead of `u8`.
