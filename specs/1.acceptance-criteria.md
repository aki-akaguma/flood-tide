# Acceptance Criteria (EARS)

This document specifies the acceptance criteria for the `flood-tide` library in the EARS (Easy Approach to Requirements Syntax) format.

## Core Functionality

### Option Parsing

- **UC-1: Short Options**
  - **While** parsing command-line arguments, **when** a short option (e.g., `-a`) is encountered, **then** the system shall recognize it as a valid option.

- **UC-2: Combined Short Options**
  - **While** parsing command-line arguments, **when** combined short options (e.g., `-abc`) are encountered, **then** the system shall treat them as multiple individual options (`-a`, `-b`, `-c`).

- **UC-3: Long Options**
  - **While** parsing command-line arguments, **when** a long option (e.g., `--verbose`) is encountered, **then** the system shall recognize it as a valid option.

- **UC-4: Abbreviated Long Options**
  - **While** parsing command-line arguments, **when** an unambiguous abbreviated long option (e.g., `--verb` for `--verbose`) is encountered, **then** the system shall recognize it as the corresponding long option.

- **UC-5: Single-Hyphen Long Options**
  - **While** parsing command-line arguments, **when** a single-hyphen long option (e.g., `-verbose`) is encountered, **then** the system shall recognize it as a valid long option.

### Option Arguments

- **UC-6: Long Option with Space**
  - **While** parsing command-line arguments, **when** a long option is followed by a value separated by a space (e.g., `--option value`), **then** the system shall associate the value with the option.

- **UC-7: Long Option with Equals Sign**
  - **While** parsing command-line arguments, **when** a long option is followed by a value separated by an equals sign (e.g., `--option=value`), **then** the system shall associate the value with the option.

- **UC-8: Short Option with Space**
  - **While** parsing command-line arguments, **when** a short option is followed by a value separated by a space (e.g., `-o value`), **then** the system shall associate the value with the option.

- **UC-9: Short Option with Adjacent Value**
  - **While** parsing command-line arguments, **when** a short option is followed by an adjacent value (e.g., `-ovalue`), **then** the system shall associate the value with the option.

- **UC-10: Optional Argument**
  - **While** an option is defined to have an optional argument, **when** the option is present without a value, **then** the system shall parse it correctly without requiring a value.

### Free Arguments

- **UC-11: Free Arguments**
  - **While** parsing command-line arguments, **when** an argument is not an option or an option's argument, **then** the system shall collect it as a free argument.

### Subcommands

- **UC-12: Subcommands**
  - **While** subcommands are defined, **when** a subcommand is present in the arguments, **then** the system shall parse the subsequent arguments based on the rules for that subcommand.

## Error Handling

- **UC-13: Invalid Option**
  - **While** parsing command-line arguments, **when** an undefined option is encountered, **then** the system shall report an "invalid option" error.

- **UC-14: Missing Option**
  - **While** a required option is not provided, **when** parsing is complete, **then** the system shall report a "missing option" error.

- **UC-15: Invalid Option Argument**
  - **While** an option's argument has an invalid format, **when** parsing that argument, **then** the system shall report an "invalid option argument" error.

- **UC-16: Unexpected Option Argument**
  - **While** an option that does not take an argument is provided with one, **when** parsing that option, **then** the system shall report an "unexpected option argument" error.

- **UC-17: Missing Option Argument**
  - **While** an option that requires an argument is provided without one, **when** parsing that option, **then** the system shall report a "missing option argument" error.

- **UC-18: Unexpected Argument**
  - **While** no more arguments are expected, **when** an argument is encountered, **then** the system shall report an "unexpected argument" error.

- **UC-19: Missing Argument**
  - **While** a required argument is not provided, **when** parsing is complete, **then** the system shall report a "missing argument" error.

- **UC-20: Invalid Subcommand**
  - **While** parsing command-line arguments, **when** an undefined subcommand is encountered, **then** the system shall report an "invalid subcommand" error.

- **UC-21: Missing Subcommand**
  - **While** a subcommand is required but not provided, **when** parsing is complete, **then** the system shall report a "missing subcommand" error.

- **UC-22: Ambiguous Option**
  - **While** an abbreviated option matches multiple long options, **when** that option is encountered, **then** the system shall report an "ambiguous option" error.

- **UC-23: Ambiguous Subcommand**
  - **While** an abbreviated subcommand matches multiple subcommands, **when** that subcommand is encountered, **then** the system shall report an "ambiguous subcommand" error.

- **UC-24: Error Collection**
  - **While** the system is configured to collect all errors, **when** multiple errors are encountered during parsing, **then** the system shall return a collection of all errors instead of just the first one.

## Platform Support

- **UC-25: `std` Environment**
  - **While** compiling for a `std` environment, **when** the library is built, **then** it shall compile successfully.

- **UC-26: `no_std` Environment**
  - **While** compiling for a `no_std` environment, **when** the library is built, **then** it shall compile successfully.

## Configuration

- **UC-27: Compile-Time Features**
  - **While** certain features are disabled at compile time, **when** the library is built, **then** the resulting binary size shall be smaller than when those features are enabled.
